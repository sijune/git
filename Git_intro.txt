GIT - 리눅스 소스코드를 관리하기 위해 개발됨.
수업을 처음부터 다 들으려 하지 말자.
우리가 지옥을 가정하자. 상상하자. 천국을 꿈꿔보자.

깃의 목적: 버전관리, 백업, 협업 (순서대로)
앞에서 머물수록 좋다.

-1. 버전관리: 또 옛날로 돌아갈 수 있을꺼 같은 느낌.
.git폴더: 버전들이 저장된다.
commit을 하면 일끝내고 퇴근하는 느낌이다. 버전을 생성한다.


-2. 백업: 언젠가 고장난다. 그것을 대비
my컴퓨터 --- github
지역저장소 --- 원격저장소

my1 --(push)--> github.com
my2 <--(pull)-- github.com
my2 --(push)--> github.com


-3. 협업: 
a가 푸쉬-> b가 풀-> b가 푸쉬 다시.의 반복 과정


git은 제품군. 다양한 제품이 있다. 
git desktop(단순), tortoisegit, sourcetree(복잡)
git cli - 명령어를 통해 제어할 수 있다.

현재까지 이민 전, 관광을 한 느낌.
